spring boot 之使用shiro对restful接口进行认证管理

发表于 2018-06-29

前言

目前的开发中，会将后台和前端严格的分离开，前端通过通过restful接口，向后台发送请求或者数据。我们之前在使用shiro的使用，当用户未登录的时候访问网站，通常会自动跳转到登录页面，访问出错的时候会跳转到错误页面。但是这样的交互设计，在restful的数据交换设计中受到了挑战。在restful中，返回的数据应该总是json格式的，因此我们需要对shiro进行一些配置。

这里是实现我们以上需求的关键，即当请求方未登录的时候，就返回未登录json信息。这主要是通过继续AdviceFilter类，并重载preHandle方法实现。

这个类允许通过很多方法实现filter中的aop的特点，比如preHandle（前置通知）,postHandle（后置通知，但是在抛异常的情况下可能不执行，）,afterCompletion（最终通知，一定会执行）。

通过以上处理，当未登录的时候就会返回false，并向response流中写入错误信息。
